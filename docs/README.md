# Документация по проектной практике

## Описание процесса выполнения практики

### 1. Подготовительный этап

На подготовительном этапе было выполнено:
- Изучение структуры задания и требований к репозиторию
- Создание git-репозитория на основе предоставленного шаблона
- Настройка локального окружения для работы с Git
- Изучение синтаксиса Markdown для оформления документации

### 2. Базовая часть

#### 2.1 Настройка Git и репозитория

- Создан личный репозиторий на GitHub
- Освоены базовые команды Git: clone, add, commit, push, branch
- Регулярно фиксировались изменения с осмысленными сообщениями коммитов

#### 2.2 Создание документов в Markdown

- Изучен синтаксис Markdown: заголовки, списки, таблицы, код, ссылки
- Подготовлена документация проекта в формате Markdown
- Созданы README файлы для всех разделов репозитория

#### 2.3 Создание статического веб-сайта

- Разработан статический сайт проекта на HTML
- Сайт включает: главную страницу, страницу "О проекте", раздел "Участники", журнал прогресса, страницу ресурсов, страницу взаимодействия с партнером
- Сайт оформлен с использованием графических материалов

### 3. Вариативная часть

#### 3.1 Выбор темы

Выбрано задание: **"Создайте простой текстовый редактор на Python"**.

Стек технологий:
- Python 3.12
- Tkinter
- ttk (themed widgets для Tkinter)

#### 3.2 Исследование и разработка

- Изучены основы создания GUI-приложений на Python
- Разработан текстовый редактор с базовым функционалом
- Проведена модификация проекта

#### 3.3 Модификация проекта

В рамках творческой модификации добавлены:
  - Создание, открытие и сохранение файлов
  - Поддержка светлой и тёмной темы оформления
  - Форматирование текста (жирный, курсив, подчёркивание)
  - Изменение шрифта и размера текста
  - Поиск и замена текста
  - Горячие клавиши
  - Статусная строка с информацией

  #### 3.4 Документирование

- Подготовлено техническое руководство по созданию текстового редактора
